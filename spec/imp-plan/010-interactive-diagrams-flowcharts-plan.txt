Implementation Plan: Interactive Diagrams and Flowcharts (spec/010-interactive-diagrams-flowcharts.txt)

## Summary

Add a new `diagram` component type to the Lecture Site Engine that renders Mermaid.js diagrams inline within lecture sections. This is purely additive: no existing components change, no schema changes to existing templates, and the new type follows the established component pattern exactly.

## Implementation Steps

### Step 1: Add Mermaid.js dependency
- Run `npm install mermaid@^10`
- Verify it appears in `package.json` dependencies

### Step 2: Add DiagramComponent type and union types
In `src/lib/lecture-template/types.ts`:
- Define `type DiagramType = "flowchart" | "sequence" | "class" | "state" | "er" | "gantt" | "pie" | "mindmap"`
- Define `type DiagramDirection = "TB" | "LR" | "BT" | "RL"`
- Define `type DiagramTheme = "default" | "dark" | "forest" | "neutral" | "base"`
- Add `DiagramComponent` interface:
  ```typescript
  export interface DiagramComponent {
    type: "diagram";
    diagram_type: DiagramType;
    title: string;
    code: string;
    direction?: DiagramDirection;
    theme?: DiagramTheme;
  }
  ```
- Add `"diagram"` to the `LectureComponentType` union
- Add `DiagramComponent` to the `LectureComponent` union

### Step 3: Add diagram validation
In `src/lib/lecture-template/validateTemplate.ts`:
- Add `"diagram"` to `supportedComponents` array
- Define `allowedDiagramTypes: DiagramType[] = ["flowchart", "sequence", "class", "state", "er", "gantt", "pie", "mindmap"]`
- Define `allowedDiagramDirections: DiagramDirection[] = ["TB", "LR", "BT", "RL"]`
- Define `allowedDiagramThemes: DiagramTheme[] = ["default", "dark", "forest", "neutral", "base"]`
- In `validateComponent`, add a block for `type === "diagram"` that:
  - Calls `requireStringFields(block, sectionTitle, errors, ["title", "code"])`
  - Validates `diagram_type` is present and in `allowedDiagramTypes`
  - Validates `code` is non-empty after trimming
  - Validates `direction` if present: must be in `allowedDiagramDirections` and only valid when `diagram_type === "flowchart"`
  - Validates `theme` if present: must be in `allowedDiagramThemes`
- In `normalizeComponent`, add a case for `data.type === "diagram"` that returns the normalized `DiagramComponent`

### Step 4: Create Diagram component
Create `src/components/lecture-kit/Diagram.tsx`:

- Accept `{ component: DiagramComponent }` props (matches Callout pattern)
- Use `useId()` for a stable diagram element ID
- Server / initial client render:
  ```
  <figure role="img" aria-label={component.title} className="lecture-component diagram-card">
    <p className="component-label">Diagram</p>
    <div className="diagram-svg-container">
      <pre><code>{component.code}</code></pre>  // no-JS fallback, always present
    </div>
    <figcaption>{component.title}</figcaption>
    <details>
      <summary>Diagram source code</summary>
      <pre><code>{component.code}</code></pre>
    </details>
  </figure>
  ```
- `useEffect` on mount:
  - Dynamically import `mermaid` (catch import error → show load failure note)
  - If `window.mermaid` already initialized, skip init; else call `mermaid.initialize({ startOnLoad: false })` — do NOT pass theme to initialize (it's global)
  - Call `const { svg } = await mermaid.render(diagramId, component.code, { theme: component.theme || 'default' })` — theme passed per-render
  - On success: replace the `<pre>` inside `.diagram-svg-container` with the SVG via `dangerouslySetInnerHTML` or a ref
  - On error: leave `<pre>` in place, show `<p className="diagram-error">Diagram could not be rendered. Showing source code.</p>`
- Use `useRef` for the container div to enable imperative SVG replacement
- The `diagramId` passed to `mermaid.render()` must be the `useId()` value

## Risks and Mitigations

1. **Mermaid.js bundle size** — Dynamic import mitigates this; it only loads when a diagram is present on the page.
2. **Mermaid.js SSR compatibility** — Mermaid requires DOM APIs. The component renders the `<pre>` fallback on the server and initializes Mermaid only in a `useEffect` on the client.
3. **Mermaid per-instance theme** — Theme is passed via `mermaid.render(id, code, config)`, not via global `initialize()`. This ensures each diagram instance uses its own theme.
4. **SVG ID collisions** — Each diagram uses `useId()` as the diagram ID passed to `mermaid.render()`, ensuring unique IDs.
5. **Source code rendered twice** — The `<details>` section is the single source of truth for source code accessibility. The `<pre>` inside `.diagram-svg-container` is the no-JS fallback; the `<details>` is always present for screen readers and keyboard users.
